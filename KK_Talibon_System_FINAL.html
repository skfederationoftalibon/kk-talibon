Database Setup
KK Profiling System — SK Talibon (one-time setup)

Connect a free Supabase database so all 25 chairmen share the same live data. Takes about 10 minutes.

1
Create a free Supabase project
Go to supabase.com → Sign up with Google → Create new project named kk-talibon → set any database password → wait ~2 min for it to start.
2
Create the database tables
In your Supabase project → click SQL Editor (left sidebar) → New query → paste the SQL below → click RUN.
-- Run this in Supabase SQL Editor
create table if not exists members (
  id uuid default gen_random_uuid() primary key,
  fn text, ln text, mn text, suffix text,
  dob text, age integer, sex text, gender text,
  civil text, brgy text, purok text, addr text,
  mobile text, email text, fb text,
  classif text, school text, year_level text,
  father text, father_c text,
  mother text, mother_c text,
  guard text, guard_c text,
  progs text, skills text, notes text,
  status text default 'Active',
  created_at timestamptz default now()
);
create table if not exists announcements (
  id uuid default gen_random_uuid() primary key,
  type text, title text, body text,
  venue text, event_date text,
  channels text[], audience jsonb,
  recipients integer, status text,
  sent_by text, sent_brgy text,
  created_at timestamptz default now()
);
create table if not exists feedbacks (
  id uuid default gen_random_uuid() primary key,
  type text, author text, brgy text,
  prog text, rating integer, comment text,
  created_at timestamptz default now()
);
create table if not exists assessments (
  id uuid default gen_random_uuid() primary key,
  member_id text, member_name text, brgy text,
  chair_name text, rating integer,
  participation text, standing text, remark text,
  created_at timestamptz default now()
);
alter table members enable row level security;
alter table announcements enable row level security;
alter table feedbacks enable row level security;
alter table assessments enable row level security;
create policy "open" on members for all using (true) with check (true);
create policy "open" on announcements for all using (true) with check (true);
create policy "open" on feedbacks for all using (true) with check (true);
create policy "open" on assessments for all using (true) with check (true);
3
Enter your Supabase credentials
In Supabase → go to Project Settings → API → copy the Project URL and anon public key.

Credentials are saved on this device only. Free tier supports up to 50,000 rows and 500MB storage.
